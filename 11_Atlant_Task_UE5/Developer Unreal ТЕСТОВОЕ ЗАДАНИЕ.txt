ТЕСТОВОЕ ЗАДАНИЕ

Оцениваться при проверке в первую очередь будут (в порядке важности): корректность работы кода, аккуратность кода, архитектура, скорость выполнения задания.

Визуальная составляющая оцениваться не будет.
Проект может быть реализован полностью на Blueprints, реализация на C++ приветствуется.

Необходимо сделать мини игру, цель которой, собрать достаточное количество нужных предметов.

Игрок появляется на карте в заданной точке (это может быть PlayerStart или аналогичный подход).
Карта представляет из себя несколько комнат. Комнаты соединены дверьми. 

В игре есть механика взаимодействия: с объектами можно взаимодействовать, нажав определенную клавишу (например "Е").

Дверь должна открываться при взаимодействии с ней игрока.

Опционально: добавить в игру точки телепорта, они связаны попарно и при взаимодействии с одной, персонажа игрока телепортирует к другой.

На карте спавнятся предметы. Точки спавна могут быть заранее определены, но тип предмета выбирается случайный.
Предмет имеет тип (например: куб, сфера или цилиндр). При взаимодействии с предметом, предмет удаляется, увеличивая соответствующий счетчик подобранных предметов (желательно отображать в UI). 

При старте игры один из типов предметов определается как требуемый для победы. С другими типами предметов взаимодействие становится невозможным. 
У всех игроков должно в UI отображаться текущее количество подобранных предметов и количество, необходимое для победы (например Cubes: 2/5).

Опционально: оповещать других игроков о том, что кто-то подобрал предмет.

При выполнении условий победы выводить на экран надпись “YOU WIN” и через N количество времени отключать игроков.
Опционально: добавить игроку возможность покинуть игру самостоятельно во время отсчета.

Время игровой сессии ограничено таймером на N секунд. Если таймер заканчивается, а условия победы ещё не выполнены - выводить на экран надпись “YOU LOSE” и через N количество отключать игроков.
Таймер необходимо отображать в UI.

Весь код необходимо сделать с учётом мультиплеера. Сервер должен быть авторитарным. Не обязательно делать игру соревновательной, например можно сделать общие очки для всех игроков. Важно сделать адекватное отображение предметов и данных для всех игроков.
